# Neural Extractor V3.0.5

V3.0.5 adds a guided `YouTube verbinden` workflow for explicit YouTube
authentication challenges.

- Public downloads still start without cookies.
- A separate Firefox profile under Neural Extractor LocalAppData handles sign-in
  and human verification without changing the user's normal Firefox profile.
- The closed profile is validated with a bounded metadata-only yt-dlp preflight
  before the original job is retried once.
- Connected sessions are reused, can be renewed in the same profile, and can be
  disconnected with exact-root and reparse-safe deletion.
- Browser arguments use `shell=False`; credentials and cookies are never placed
  on the command line or in logs.
- Blind Chrome/Edge/Brave/Firefox cycling is disabled in the normal guided flow;
  `cookies.txt` and manually enabled legacy browser fallback remain advanced options.
- A PO Token provider interface and diagnostics are included. No provider is
  bundled, and V3.0.5 does not accept manual PO Tokens or falsely claim provider support.
- V3.0.4 updater ownership, manifest, SHA-256, backup, startup confirmation, and
  rollback behavior remain unchanged.

Authenticated automated downloading can carry account restrictions. Use the
connection only when required; a secondary account may be preferable.
