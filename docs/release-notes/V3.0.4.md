# Neural Extractor V3.0.4

V3.0.4 fixes false “Another updater process owns this installation” failures by correcting the GUI-to-helper transaction ownership handoff used by the packaged Windows application.

- Adds PID plus process-creation identity validation so PID reuse and exited owners are handled safely.
- Adds target-scoped stale updater-state recovery without deleting validated live ownership.
- Preserves rejection of genuinely concurrent updater processes targeting the same installed EXE.
- Preserves mandatory SHA-256 and file-size verification before replacement.
- Preserves backup-before-replacement, transaction-bound startup confirmation, and automatic rollback.
- Preserves all V3.0.3 YouTube reliability fixes, including no-cookie-first attempts, controlled cookie fallback, worker isolation, timeouts, cancellation, and process-tree cleanup.

V3.0.2 and V3.0.3 users may need to install V3.0.4 manually once because the updater inside those installed versions contains the handoff defect. Future compatible updates from V3.0.4 should use the repaired updater; the V3.0.4-to-V3.0.5 path still requires real field validation before it is described as proven on an owner’s PC.

The EXE remains unsigned, so Windows SmartScreen may warn. Download Neural Extractor only from the official `AegisAI-Dev/NeuralExtractor` GitHub Release and verify the published checksum when possible.
