# Neural Extractor V3.0.7

V3.0.7 is a focused Windows Unicode hotfix for the isolated yt-dlp worker. It
keeps the responsive interface, managed-browser authentication, and updater
guarantees from V3.0.6.

- Worker protocol frames are now written and read explicitly as UTF-8 instead of
  depending on the active Windows console code page.
- YouTube titles and log messages containing U+FF5C FULLWIDTH VERTICAL LINE and
  other characters unavailable in CP1252 no longer crash the worker transport.
- Dedicated Google Chrome and Firefox authentication were already valid. This
  hotfix repairs the worker transport and log forwarding that runs after
  authentication and format selection succeed.
- Unicode titles, subtitles, output paths, emoji, and non-Latin text remain
  preserved; filename restriction or character removal is not used as a fix.
- The responsive GUI, isolated managed Chrome and Firefox profiles, stale-browser
  recovery, Stop/Cancel handling, timeouts, and owned process-tree cleanup are
  preserved.
- Automatic update installation, transaction-bound startup confirmation, backup,
  and rollback behavior are unchanged.

The EXE remains unsigned, so Windows SmartScreen may warn. YouTube can change its
authentication, client, format, and challenge requirements without notice, so not
every video is guaranteed to work. Download Neural Extractor only from the official
`AegisAI-Dev/NeuralExtractor` GitHub Release and verify the published checksum when
possible.
