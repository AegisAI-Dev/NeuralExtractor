# Neural Extractor V3.0.5 — Guided YouTube Connection

## Purpose

V3.0.5 replaces manual cookie export as the normal response to YouTube's
“Sign in to confirm you're not a bot” challenge. Public downloads still begin
without authentication. Only an explicit authentication-required result opens
the `YouTube verbinden` assistant.

The assistant launches Firefox with a Neural Extractor-owned profile, lets the
user complete sign-in or CAPTCHA directly in Firefox, verifies the closed
profile with a bounded metadata-only yt-dlp request, and retries the original
job once. Future authenticated attempts reuse the same profile. `cookies.txt`
remains available only as an advanced compatibility fallback.

## Dedicated profile architecture

The Windows profile is fixed at:

```text
%LOCALAPPDATA%\NeuralExtractorV3\youtube\firefox-profile\
```

It is current-user writable, outside the installation directory, outside
Mozilla's normal profile directories, stable across app updates, and never
registered in `profiles.ini`. Firefox is launched directly with an argv list
equivalent to:

```text
firefox.exe -no-remote -profile <managed-profile> -url <youtube-url>
```

The launch uses `shell=False`. Neural Extractor tracks only the process it
started, including PID and process-creation identity while the app is running.
It never terminates an ordinary Firefox session and does not force-close the
login browser on shutdown.

Firefox discovery checks a previously validated path, Windows App Paths, and
the standard 64-bit and 32-bit Program Files locations. A candidate must be an
existing PE executable named `firefox.exe`. If discovery fails, the user can
select `firefox.exe`; the validated path is remembered. Firefox is never
silently downloaded or installed.

## Connection and verification flow

1. The first download attempt uses `auth=none`.
2. An explicit authentication challenge pauses only that job.
3. The assistant explains the isolated profile, privacy boundary, and account risk.
4. `Firefox openen` opens the requested video in the managed profile.
5. The user signs in, completes any challenge, confirms playback, and closes Firefox.
6. `Opnieuw controleren` checks that the profile is closed and unlocked.
7. Neural Extractor confirms `cookies.sqlite` exists and queries only cookie
   names and domains for YouTube/Google session markers. Cookie values are never selected.
8. A 90-second total / 45-second inactivity bounded yt-dlp subprocess performs
   metadata discovery with structured `cookiesfrombrowser=("firefox", profile)` options.
9. Success records the verification time and retries the original job once.
10. Other queued jobs can continue while the user completes the browser step.

The presence of `cookies.sqlite` alone is never considered proof of a working
connection. Verification distinguishes authentication rejection, expired
cookies, profile locking, network timeout, account access restrictions, format
failures, and PO Token requirements.

## Lifecycle

The persisted state is one of:

```text
not_configured, firefox_missing, profile_ready, waiting_for_login,
browser_open, verifying, connected, expired, locked, invalid,
disconnected, error
```

Persisted settings contain only the managed profile path, state, last verified
time, sanitized failure reason, and optional validated Firefox executable path.
Browser PID and creation identity are transient in-memory process metadata.

When yt-dlp reports rotated or invalid cookies, the state becomes `expired` and
the UI offers `Verbinding vernieuwen`. Renewal opens the same profile; it does
not create a new profile or fall back to the normal Firefox profile.

## Disconnect safety

`YouTube loskoppelen` requires confirmation. Deletion proceeds only if Firefox
is closed and the candidate is exactly the fixed managed profile beneath the
expected Neural Extractor LocalAppData root. Root deletion, traversal, symlink
or junction/reparse escapes, and a locked profile are rejected. Every reparse
point inside the tree is rejected before recursive deletion.

Only the dedicated profile is removed. Normal Firefox profiles, `profiles.ini`,
and `cookies.txt` are never touched. If Windows prevents removal, the UI keeps
recoverable state and reports the exact manual cleanup path.

## Privacy and threat model

- Session cookies remain in Firefox's local profile storage; no
  plaintext `cookies.txt` copy is created by the guided flow.
- Neural Extractor never asks for, reads, automates, intercepts, or stores the
  user's Google password.
- Cookie values, account email addresses, Google identifiers, and tokens are
  not displayed or logged.
- The full managed profile path is redacted from ordinary diagnostics and
  user-facing connection messages.
- The profile is never uploaded, synchronized, added to crash reports, copied
  into the installation directory, exposed to the updater, or included in
  updater executable backups.
- Only normal Firefox/yt-dlp requests to YouTube receive the session cookies.
- Authenticated automated downloading can lead to temporary or permanent
  account restrictions. Authentication should be used only when required; a
  secondary account may be preferable.

## PO Token provider decision

No provider is bundled in V3.0.5. The internal provider interface and complete
failure taxonomy are implemented, but status is accurately reported as
`provider_missing`. Manual PO Tokens, hardcoded tokens, remote token services,
and downloaded executable code are not accepted.

The review used the official yt-dlp PO Token Guide as of 2026-07-20:

- `Brainicism/bgutil-ytdlp-pot-provider` 1.3.1 is a featured provider maintained
  by a yt-dlp maintainer. It is GPL-3.0, requires Node.js 20+ or Deno, includes
  JavaScript/npm dependencies (including native `canvas`), and either starts a
  local HTTP service (default port 4416) or spawns a token-generation script.
  License impact, pinned transitive artifact hashes, native Windows/PyInstaller
  behavior, service lifecycle, and bundled-Node integration were not proven to
  the standard required for this release.
- `coletdjnz/yt-dlp-getpot-wpc` is a featured MIT-licensed experimental provider
  maintained by a yt-dlp core maintainer. It requires Chrome/Chromium and
  `nodriver`, launches an automated Chromium browser during yt-dlp execution,
  and is not compatible with the dedicated non-automated Firefox trust boundary
  chosen for V3.0.5.

Because neither candidate completed the required license, dependency, network,
local-service, remote-code, exact-hash, Windows x64, bundled-Node, PyInstaller,
and updater-isolation review, bundling would be unsafe to claim. PO Tokens are a
separate playback-attestation mechanism: they do not replace account cookies,
solve private/member access, repair expired sessions, or guarantee bypass of a
bot check.

References:

- https://github.com/yt-dlp/yt-dlp/wiki/PO-Token-Guide
- https://github.com/Brainicism/bgutil-ytdlp-pot-provider
- https://github.com/coletdjnz/yt-dlp-getpot-wpc

## Known limitations

- Firefox must be installed or selected manually; Neural Extractor does not install it.
- YouTube may still reject an account, IP address, region, age, membership, or
  private-video request after a valid session is connected.
- Some clients and formats require a PO Token; V3.0.5 reports this precisely but
  does not generate one.
- Verification requires the dedicated Firefox window to be fully closed.
- YouTube behavior is server-controlled and may change independently of the app.

## Manual owner field test

1. Install the local V3.0.5 test build.
2. Ensure no valid `cookies.txt` is loaded.
3. Attempt a video that triggers “Sign in to confirm you're not a bot.”
4. Confirm Neural Extractor offers `YouTube verbinden`.
5. Open the dedicated Firefox profile.
6. Confirm the normal Firefox profile is unchanged.
7. Sign into YouTube in the dedicated window.
8. Complete any verification.
9. Play the requested video briefly.
10. Close the dedicated Firefox window.
11. Return to Neural Extractor and choose `Opnieuw controleren`.
12. Confirm the session is verified.
13. Confirm the original job resumes once.
14. Confirm subsequent authenticated downloads reuse the profile.
15. Use `Verbinding vernieuwen` and confirm the same profile reopens.
16. Use `YouTube loskoppelen` and confirm deletion.
17. Confirm only the managed LocalAppData profile was deleted.
18. Confirm normal public videos still begin with `auth=none`.
19. Confirm Stop/Cancel and queue recovery still work.
20. Confirm the V3.0.4 updater can detect V3.0.5 and complete its normal verified handoff.

Do not use a primary account for this test if the account-risk warning is unacceptable.
